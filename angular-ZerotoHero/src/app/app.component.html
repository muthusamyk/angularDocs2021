  • It's where you write your bind code
  • Binds the View and model

Data Binding

  • Communication b/w typescript Component and the html template
  • types of data binding :
    • String Interpolation --> Send data from TypeScript component to Html template
    • Property Binding [] --> we can directly bind the values to the property ex: [src] for image tag
    • Event Binding () --> pass the data from html template to typescript ex: input value pass to typescript from html... Also used for mouse movement keyup clicks and etc
    • Two way Binding [()] --> By using ngModel to bind data both direction

Directives

  • It helps to attach behaviour to your html DOM.
    • Structural Directives --> Modify the DOM
    • Attribute Directives --> Modify the Properties of the DOM

Services

  • Typescript class used for sharing data or functionality.
  • No need to specify any decorator for services. like ex:: @Component @NgModule
  • Need to provide the service either component or in the module
    • When we include into the Component then this component and the child component get the same instance of that sevice
    • When we include into the root Module then the entire appln will have one instance of that sevice
  • Services can be injected into another services... But inthat case its necessary to add @injectable decorator in the service to receiving the other service
  • newer version of angular suggested to add @injectable decorator to all the services...
  • uses of services
    • code reuse
    • Communication b/w Components
